## 2026-03-07: 索引機能 + ローカルLLM構想

### 索引/インデックス機能（確度A → v1.0〜v2.0）

**発端**: 複数フォルダ・深い階層を扱うと「どこに何があったか」が分からなくなる問題。

**設計方針:**
- 一度開いたファイルのメタデータ（パス、見出し一覧、先頭サマリ）をキャッシュ
- `~/Library/Application Support/md-tool/` に保持 → ユーザーフォルダは汚さない（No Database整合）
- v1.0: 基本的な一覧・検索・ナビゲーション
- v2.0: トピック分類

**評価**: コンセプト整合性◎、実用性◎、競合にない差別化要素。

### ローカルLLM統合（確度C → v3.0以降）

**発端**: ローカルモデルでMDファイル群をベースにした簡易RAGができれば便利。

**設計方針:**
- アプリにモデルは同梱しない（軽量維持）
- 設定画面からモデル選択・ダウンロード（VoiceInk的UX）
- 自前サーバーからのダウンロードに対応
- 推論ランタイム: llama.cpp / MLX（Mac最適化）

**段階的アプローチ:**
1. セマンティック検索（エンベディングのみ、生成回答なし）
2. 簡易RAG対話（MDファイル群ベースの質問応答）
3. 要約・分類

**重要な意思決定:**
> v1.0〜v2.0でコア機能を固めてから着手する。アプリが安定するまでローカルLLM機能は実装しない。

**レビュー時の懸念点:**
- 「何もしない美学」からの逸脱 → プラグイン的位置づけで分離
- アプリサイズ目標（50MB）との矛盾 → オンデマンドダウンロードで解決
- No Database矛盾 → Application Support内に閉じれば問題なし
- 軽量モデルの品質期待値 → まずセマンティック検索に絞り、生成は後回し
