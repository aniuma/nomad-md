## 1. プロジェクト概要

特定のデータベース（倉庫）に依存せず、MacのFinder上のフォルダ構造をそのまま「サイドバー」に投影し、1つのウィンドウ内で複数のマークダウンファイルを軽快にプレビュー・管理するネイティブMacアプリケーション。

**アプリ名**: md-tool（仮称）

---

## 2. コア・コンセプト

| コンセプト | 説明 |
|-----------|------|
| **No Database** | 独自の管理ファイルや独自の保存形式を持たない。ユーザーのフォルダに一切ファイルを書き込まない |
| **Direct Access** | ユーザーが指定した任意のローカルフォルダをそのまま読み込む。「インポート」不要 |
| **One-Window Experience** | ファイルごとにウィンドウを開かず、単一画面内で完結する |

**設計哲学**: 「何もしない美学」— DBを作らない、設定を書き込まない、余計な機能を足さない。

## 4. 技術アーキテクチャ

### 4.1 技術スタック

| カテゴリ | 選定 | 理由 |
|---------|------|------|
| UIフレームワーク | **SwiftUI + AppKit ハイブリッド** | SwiftUIの宣言的UIとAppKitの成熟した機能を両立 |
| 最小OS | **macOS 14 Sonoma** | `@Observable`マクロ対応、SwiftUI性能改善 |
| 言語 | **Swift 5.9+** | - |
| アーキテクチャ | **MVVM + @Observable** | 本アプリの規模にはTCAは過剰 |
| MDパーサー | **swift-markdown** (Apple公式) | GFM対応、ASTベース、thread-safe |
| MDレンダラー | **WKWebView + カスタムCSS** | CSSスタイリング自在、highlight.js等を活用可能 |
| ファイル監視 | **FSEvents + DispatchSource** | FSEvents=ディレクトリ全体、DispatchSource=開いているファイル |
| 配布 | **初期: 公証付き直接配布 → 将来: App Store** | 初期は非サンドボックスで開発速度優先 |

### 4.2 フレームワーク構成の詳細

| UIパーツ | 技術 |
|---------|------|
| メインレイアウト | SwiftUI `NavigationSplitView` で3カラム構成 |
| ファイルツリー | SwiftUI `OutlineGroup` → 性能問題時は `NSOutlineView` にフォールバック |
| プレビュー | `WKWebView` を `NSViewRepresentable` でラップ |
| エディタ | `NSTextView`（TextKit 2、シンタックスハイライト付き） |
| メニュー/ツールバー | SwiftUI `.commands` / `.toolbar` |

### 4.3 データフロー

```
FileSystem ──FSEvents──→ FileSystemService
                              │
                              ▼
                     DirectoryModel (AST)
                              │
                              ▼
                   SidebarViewModel (@Observable)
                     │              │
                     ▼              ▼
              SidebarView    PreviewViewModel (@Observable)
                                    │
              DispatchSource ──→ MDParser (swift-markdown)
                                    │
                                    ▼
                              PreviewView (WKWebView)
```

**主要コンポーネント:**
- **FileSystemService**: FSEvents監視、ディレクトリスキャン、ブックマーク管理
- **DirectoryModel**: ファイルツリーの`Identifiable` + `Hashable`な値型
- **SidebarViewModel**: ツリー状態、選択状態、フィルタリング
- **PreviewViewModel**: MD解析、HTML生成、レンダリング状態
- **BookmarkManager**: セキュリティスコープブックマークの永続化

### 4.4 ファイル監視戦略

| 監視対象 | 方式 | 用途 |
|---------|------|------|
| ディレクトリツリー | FSEvents | ファイル追加/削除/リネームの検知 |
| 開いているファイル | DispatchSource | ファイル内容変更の検知 |

### 4.5 パフォーマンス戦略

| 課題 | 対策 |
|------|------|
| 大量ファイル（1000+） | 遅延スキャン（展開時に子ディレクトリを読み込み）、`.git`/`node_modules`等の除外 |
| MDレンダリング | 300msデバウンス、HTMLキャッシュ（パス+更新日時キー）、WKWebView再利用 |
| メモリ管理 | `NSCache`による自動パージ、プレビュー対象のみメモリ保持 |
| 大規模フォルダ初回スキャン | バックグラウンド非同期 + プログレスインジケータ |
| スリープ復帰後 | FSEventsフォールバック（フルスキャンをトリガー） |

### 4.6 サンドボックスとファイルアクセス

| 配布方式 | サンドボックス | ファイルアクセス |
|---------|-------------|--------------|
| **直接配布（推奨・初期）** | 不要 | Hardened Runtime + 公証で自由アクセス |
| App Store（将来） | 必須 | セキュリティスコープブックマーク必須 |

**App Store対応時の実装:**
1. `NSOpenPanel` でユーザーにフォルダ選択を促す
2. `URL.bookmarkData(options: .withSecurityScope)` でブックマーク保存
3. アプリ再起動時に `URL(resolvingBookmarkData:options:.withSecurityScope)` で復元
4. `startAccessingSecurityScopedResource()` / `stopAccessingSecurityScopedResource()` で管理

### 5.11 ビジュアルデザイン方針

#### 5.11.1 デザインコンセプト

**「紙を開くような軽さ」と「Macの息遣い」**

- 起動からプレビュー表示まで、紙のノートを開くような自然さと速度感
- macOSネイティブの質感を最大限活かし、「Macの中に自然に存在する」佇まい
- 装飾は引き算。要素が少ないほど美しい

#### 5.11.2 タイポグラフィ

| 用途 | フォント | サイズ | 備考 |
|------|---------|--------|------|
| プレビュー本文 | SF Pro Display | 16-18px | 行間1.5-1.6倍、行幅50-75文字（66文字が黄金比） |
| プレビュー見出し | SF Pro Display Semibold | h1:28px, h2:24px, h3:20px, h4:18px, h5:16px, h6:14px | 見出し間の余白で階層を表現 |
| コードブロック | SF Mono | 14px | リガチャ有効 |
| エディタ | SF Mono | 14px | 等幅で編集体験を最適化 |
| サイドバー | SF Pro Text | 13pt（ルート14pt semibold） | 既存5.5仕様に準拠 |

#### 5.11.3 カラーパレット

**ライトモード:**

| 用途 | カラー | 備考 |
|------|--------|------|
| プレビュー背景 | `#FFFBF7` | 温かみのある米色。純白より目に優しい |
| プレビュー文字 | `#1D1D1F` | Apple標準に近い深い黒 |
| エディタ/サイドバー背景 | `NSColor.windowBackgroundColor` | システム標準に追従 |
| アクセント | macOSシステムアクセントカラー | ユーザー設定を尊重 |

**ダークモード:**

| 用途 | カラー | 備考 |
|------|--------|------|
| プレビュー背景 | `#2D2D2D` | 純黒より柔らかいダークグレー |
| プレビュー文字 | `#E5E5E7` | 眩しすぎない白 |
| エディタ/サイドバー背景 | `NSColor.windowBackgroundColor` | システム標準に追従 |

- セマンティックカラー（`NSColor`）を基本とし、ライト/ダーク自動追従
- プレビュー領域のみカスタムカラーを使用（テーマで切替可能）

#### 5.11.4 アイコン

- **SF Symbols 7** で100%統一。サードパーティアイコンは使用しない
- サイズ: 16x16pt（サイドバー）、14x14pt（ツールバー）、20x20pt（空状態の装飾）
- レンダリングモード: モノクロ（システムアクセントカラーに追従）

#### 5.11.5 密度・余白

- Bearを参考に「呼吸感のある」バランスを追求
- サイドバー行高: 24pt（既存仕様）、ファイルリスト行高: 44pt
- プレビュー: 左右パディング40-60pt、段落間余白1.2em
- 情報密度より可読性を優先。詰め込まない

#### 5.11.6 アニメーション

控えめ・機能的なアニメーションのみ。装飾的なアニメーションは入れない。

| 対象 | 種類 | 時間 |
|------|------|------|
| サイドバー折りたたみ/展開 | スライド | 0.25秒 |
| モード切替（プレビュー↔編集） | クロスフェード | 0.15秒 |
| ファイル切替 | なし（即時表示） | - |
| テーマ切替 | フェード | 0.2秒 |

- `NSAnimationContext` / SwiftUI `.animation(.easeInOut)` を使用
- `Reduce Motion` 設定時はすべてのアニメーションを無効化

#### 5.11.7 Liquid Glass（将来対応）

## 6. 機能要件

### A. フォルダ管理

#### A-1. フォルダの登録・解除

> **ユーザーストーリー**: ユーザーとして、任意のフォルダをアプリに登録して、その中のMarkdownファイルをサイドバーで一覧・管理したい。

**受入基準:**
- [ ] Finderからフォルダをドラッグ&ドロップしてサイドバーに登録できる
- [ ] メニューバーまたは`[+]`ボタンからNSOpenPanelでフォルダを選択・登録できる
- [ ] 登録フォルダはアプリ再起動後も保持される（セキュリティスコープブックマークで永続化）
- [ ] フォルダを右クリック→「登録解除」で一覧から除外できる（ディスク上のファイルには影響なし）
- [ ] 登録解除時に確認ダイアログを表示する

#### A-2. 複数ルートフォルダの同時管理

> **ユーザーストーリー**: ユーザーとして、仕事用・個人用・プロジェクト別など、複数のルートフォルダを同時にサイドバーに表示して横断的にアクセスしたい。

**受入基準:**
- [ ] 複数のルートフォルダをツリー形式で同時表示できる
- [ ] 各ルートフォルダは折りたたみ/展開が可能
- [ ] 表示順序をドラッグ&ドロップで変更できる
- [ ] 登録上限は設けない（パフォーマンス目標: 10フォルダ以上）

#### A-3. ファイル変更の自動検知

> **ユーザーストーリー**: ユーザーとして、Finderや外部エディタでファイルを変更した際に、サイドバーが自動的に最新状態に更新されてほしい。

**受入基準:**
- [ ] FSEventsでファイルシステムの変更をリアルタイム監視する
- [ ] ファイルの追加・削除・リネーム・移動が1秒以内に反映される
- [ ] プレビュー中のファイルが外部変更された場合、自動で再レンダリングする
- [ ] 監視のCPU負荷は通常時0.5%以下

#### A-4. ファイルフィルタリング

**受入基準:**
- [ ] デフォルトで `.md` / `.markdown` のみ表示
- [ ] 対象ファイルが0件のフォルダは非表示
- [ ] 設定で対象拡張子を追加可能（`.txt`, `.mdx`等）
- [ ] `.git`, `.obsidian`, `node_modules` 等はデフォルト非表示
- [ ] 除外パターンを設定可能
- [ ] フィルタのON/OFFをトグル可能（全ファイル表示モード）

### B. ファイルプレビュー

#### B-1. Markdownレンダリング

**受入基準:**
- [ ] CommonMark仕様に完全準拠
- [ ] GFM（GitHub Flavored Markdown）拡張に対応
- [ ] ファイル選択→プレビュー表示: 200ms以内（50KB以下のファイル）
- [ ] WKWebViewベースで表示
- [ ] CSSテーマを選択可能（ライト/ダーク、最低2種）
- [ ] macOSのダークモード切替に自動追従

#### B-2. 対応Markdown要素

**MVP（必須）:**
- 見出し（h1〜h6）、段落・改行、強調（太字・斜体・取り消し線）
- リスト（順序付き・順序なし・ネスト）、リンク、画像
- コードブロック（シンタックスハイライト付き）、インラインコード
- 引用（blockquote）、水平線、テーブル（GFM）、チェックリスト

**v1.0（拡張）:**
- 数式レンダリング（KaTeX）、Mermaidダイアグラム
- 脚注、定義リスト、目次の自動生成（TOC）

**v2.0（高度）:**
- YAML Front Matterの解析・表示
- カスタムコンテナ（admonition / callout）
- 埋め込みコンテンツ（oEmbed）

#### B-3. 画像の表示

**受入基準:**
- [ ] 相対パス・絶対パス・リモートURL（https://）のいずれも表示可能
- [ ] 対応形式: PNG, JPEG, GIF, SVG, WebP
- [ ] 画像が見つからない場合は代替テキスト + プレースホルダーを表示
- [ ] 画像サイズがウィンドウ幅を超える場合は自動縮小

#### B-4. リンクの処理

**受入基準:**
- [ ] `.md`への相対リンク → アプリ内でそのファイルを開く
- [ ] `http://` / `https://` → デフォルトブラウザで開く
- [ ] アンカーリンク（`#heading`）→ プレビュー内スクロール
- [ ] リンク先が存在しない場合 → ツールチップで警告

### C. ファイル編集

#### C-1. テキスト編集

**受入基準:**
- [ ] `Cmd+E`でプレビュー↔編集のモード切替
- [ ] 基本操作: 選択、コピー、ペースト、カット、全選択
- [ ] 行番号表示（ON/OFF切替可能）
- [ ] Markdown構文ハイライト
- [ ] タブキーでスペース挿入（2/4スペース設定可能）
- [ ] 分割表示（サイドバイサイド）- v1.0以降

#### C-2. 保存方式（ハイブリッド）

**受入基準:**
- [ ] 自動保存をデフォルト有効（入力停止後2秒）
- [ ] 自動保存ON/OFFを設定で切替可能
- [ ] OFF時は `Cmd+S` で手動保存
- [ ] 未保存変更にインジケータ（ドット等）を表示
- [ ] アプリ終了時に未保存変更があれば確認ダイアログ

#### C-3. Undo/Redo

- [ ] `Cmd+Z` / `Cmd+Shift+Z`
- [ ] ファイルごとに独立管理、100ステップ以上
- [ ] 意味のある操作単位でグルーピング

#### C-4. 文字エンコーディング

- [ ] UTF-8デフォルト、BOM付きUTF-8も対応
- [ ] UTF-8以外検出時はステータスバーに警告
- [ ] 改行コード（LF/CRLF/CR）は元のコードを維持
- [ ] 新規ファイル: UTF-8（BOMなし）+ LF

### D. 検索・フィルタ

#### D-1. ファイル名検索（クイックオープン）

- [ ] `Cmd+P` でパレット表示
- [ ] インクリメンタルサーチ + ファジーマッチング
- [ ] 検索結果にファイルの所属フォルダパスを表示
- [ ] 最近開いたファイルを優先表示

#### D-2. 全文検索

- [ ] `Cmd+Shift+F` で全文検索パネル表示
- [ ] 登録済み全フォルダを横断検索
- [ ] 結果をファイルごとにグルーピング + 前後コンテキスト表示
- [ ] 大文字小文字区別ON/OFF
- [ ] 正規表現検索（v1.0以降）
- [ ] 非同期実行、UIをブロックしない

## 8. エッジケース・例外処理

### 8.1 外部でファイルが削除された場合

| 状態 | 振る舞い |
|------|----------|
| サイドバーに表示中（非選択） | 自動的に除去 |
| プレビュー表示中 | 「このファイルは削除されました」バナー表示。内容は保持 |
| 編集中（未保存変更あり） | 警告ダイアログ → 「名前を付けて保存」or「破棄」 |

### 8.2 外部でファイルが変更された場合

| 状態 | 振る舞い |
|------|----------|
| プレビュー表示中（編集なし） | 自動で最新内容に再レンダリング |
| 編集中（未保存変更あり） | コンフリクトダイアログ: 「外部変更を取込」/「自分の変更を維持」/「差分を確認」 |
| 編集中（変更なし） | サイレントに最新内容を反映 |

### 8.3 権限エラー

- 登録時にアクセス権なし → エラーダイアログ
- 後から権限変更 → フォルダアイコンに警告マーク + ツールチップ
- ブックマーク期限切れ → 再選択を促すダイアログ

### 8.4 その他

| ケース | 振る舞い |
|-------|----------|
| 壊れたMarkdown | パーサーが解釈できない部分はプレーンテキストとして表示（エラーにしない） |
| バイナリファイルに.md拡張子 | 「テキストファイルとして読み取れません」メッセージ |
| シンボリックリンク | デフォルトで追従。循環検出（最大深度20）。壊れたリンクはグレーアウト |
| 大ファイル: 〜1MB | 通常レンダリング |
| 大ファイル: 1MB〜10MB | 警告バナー表示後にレンダリング |
| 大ファイル: 10MB〜50MB | 先頭部分のみ表示。編集はプレーンテキストに制限 |
| 大ファイル: 50MB以上 | 「外部エディタでの編集を推奨」メッセージ |

| 機能 | 概要 | 検討ポイント |

| レビューUI | セクション/段落ごとの承認・却下・コメント機能 | 実装コスト大。コメントデータの保存先がNo Databaseと矛盾する可能性 |
| Markdownリンター | 構文の一貫性チェック（リスト記号統一、空行ルール等）。設定不要のsensible defaults | ルール設計。厳しすぎるとユーザー体験を損なう |
| プレゼンモード | H2区切りでスライド的に表示。AI生成アウトラインをそのままプレゼンに | 実装コスト。レイアウト制御の複雑さ |
| 音声読み上げ | macOS AVSpeechSynthesizerで選択セクションを読み上げ。長文レビュー用 | VoiceOverとの棲み分け。OS標準機能との重複 |

### 10.4 将来構想: ローカルLLM統合（v3.0以降）

> **前提**: v1.0〜v2.0のコア機能が安定した後に着手する。アプリの基盤が固まるまでは実装しない。

**段階的アプローチ:**

| 段階 | 機能 | 概要 |
|------|------|------|
| Step 1 | セマンティック検索 | エンベディングのみ。質問→該当セクションを表示。生成回答なし |
| Step 2 | 簡易RAG対話 | MDファイル群をコンテキストに、ローカルモデルで質問応答 |
| Step 3 | 要約・分類 | ファイルの自動要約、トピック分類の提案 |

**モデル管理の設計方針:**
- アプリ初期状態にはモデルを同梱しない（軽量を維持）
- 設定画面からモデルを選択・ダウンロード（VoiceInk等と同様のUX）
- モデルファイルは `~/Library/Application Support/md-tool/models/` に保存
- 自前リポジトリからのダウンロードに対応（ホスティングはユーザー提供サーバー）
- 推論ランタイムは llama.cpp / MLX 等のMac最適化ライブラリを想定

**コンセプトとの整合性:**
- No Database: エンベディングインデックスは `Application Support` 内に閉じる。ユーザーフォルダには一切書き込まない
- 軽量性: モデル未ダウンロード時はゼロコスト。ダウンロード後もオンデマンド推論
- ローカル完結: クラウドAPI不要。プライバシー保護

### 10.5 採用判断基準

| 判断軸 | 重み | 説明 |
|--------|------|------|
| コンセプト整合性 | 最重要 | No Database / Direct Access / One-Window と矛盾しないか |
| 実装コスト | 高 | MVP〜v2.0のスケジュールに収まるか |
| 差別化度 | 中 | 競合ツールにない独自価値を提供するか |
| 段階的実装可否 | 中 | 最小限の機能から段階的に拡張できるか |
